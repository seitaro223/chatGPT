# REVIEW_PROTOCOL.md

第三者レビュー手順書（下層データ照合あり）

このファイルは、上澄み判断ログを Claude Code に第三者レビューさせるための定型手順である。
起動するときは、対象ファイルを指定して「REVIEW_PROTOCOL.md の手順で {対象md} をレビューして」と頼む。

---

## 0. 役割の固定

レビュアーは、人間の研究判断を批判的に検証する第三者である。
最終的な採用 / 保留 / 撤回は決めない。指摘・疑問・対立仮説の提示までが役割。
人間の決定権を奪わない。レビュアーは「土台と推論に独立な目を入れる」係に徹する。

---

## 1. 対象

- レビュー対象（上澄み判断ログ）：起動時に指定する md ファイル一本。
  - 例：`20260615_H52Q_working_log_r18.md`
- 下層データ：リポジトリ内の MATLAB 出力およびデータ整理ファイル群（量が多い。全部読んでよい）。
  - **主体は xlsx**（md・csv も混在しうる）。
  - xlsx は1ファイルに複数シートが入る。**ファイル名からは中身が分からないシート**
    （例：`02_xeq_recalc`、`*_Fform_check` 等）が下層に埋もれている前提で、
    全ファイルの**全シート**を列挙してから読むこと。シート単位で見落とさない。

---

## 2. 最初に必ずやること（カバレッジ宣言）

1. リポジトリ全体を**再帰的に**列挙する（サブフォルダを含む）。
   - xlsx は**シート単位**で列挙する（ファイル名だけでなく、各ファイルの全シート名を出す）。
2. 各フォルダ・主要ファイルについて「読んだ / 読まなかった」を表にする。
3. 読まなかったものは、その理由を明記する（無関係と判断した根拠）。
4. このカバレッジ表を、レビュー本体の**前**に出力する。

→ これにより、人間が「見落とされたフォルダ」に気づける。
   「全部見たはず」を暗黙に信じない。読んでいない箇所は自己申告させる。

---

## 3. レビュー本体（判断ごとに3点セット）

対象 md から主要な判断を抽出し、判断ごとに以下を返す。

1. **前提チェック**
   - その判断の根拠になっている下層の数字を特定する（どのファイルのどの値か）。
   - その数字が下層データと整合しているか。
   - 汚染・足しすぎ（DNB位置より下流の面積混入等）・手動オフセット・
     条件不一致（short-long のマッチング条件ズレ等）がないか。
   - **xlsx 特有の汚染源を狙って見る**：
     - SUM/積分範囲のラベルと実際の参照範囲の食い違い
     - 別列の値の紛れ込み（例：`F_form_old` 列の値がそのまま使われている）
     - セルへの手動オフセット（例：`= ... - 0.01` のような直書き補正）
     - 同じ汚染値が別シート・別ファイルへ伝播していないか

2. **推論チェック**
   - 根拠から結論への運びに、飛躍や交絡の見落としがないか。
   - 特に以下を厳しく見る：
     - level（討論対象の絶対水準）と gradient（変化率・傾き）の混同
     - result quantity（出口クオリティ等の結果量）を入力・閉包に使っていないか
     - 強く共変する軸（Hsub と L/D、F_form と L/DH 等）を独立効果と誤読していないか

3. **対立仮説**
   - 人間が検討していない、または弱くしか潰していない別の説明軸はないか。
   - あれば、それをどの下層データで検証できるかも添える。

---

## 4. 出力形式

カバレッジ表（セクション2）のあとに、判断ごとの表を出す。

| 判断ID | 人間の結論 | 前提チェック結果（根拠ファイル名つき） | 推論チェック結果 | 未検討の対立仮説 | 所見（同意 / 疑問 / 要再検証） |

- 所見は散文の長文にしない。フラグと一行コメントで。
- すべての所見に、根拠にした下層ファイル名を必ず添える。

---

## 5. 禁則

- 採用 / 保留 / 撤回を決めない。指摘と疑問の提示まで。
- 補正式をフィットして「これが正解」と提示しない。
- 下層を読まずに上澄みだけで所見を書かない（必ず根拠データに当たる）。
- 結論の言い切りをしない。「下層に支持される / されない / 判断材料不足」の三択で留める。

---

## 6. このプロトコルの位置づけ

- これはレビューであって、最終判断ではない。
- レビュアーの所見が出たら、その所見自体の妥当性を人間（＋チャット側 Claude）で再度読む。
- 三者構成：
  - 下層を全部読む係 → Claude Code（このプロトコル）
  - 所見と元判断を突き合わせて議論する相手 → チャット側 Claude
  - 採用 / 保留 / 撤回を決める → 人間
