# Claude Code検証依頼：x_eq の配線（単管 noF1/F1・バンドル noF1/F1）の事実確認

これは確認形の依頼ではない。**実コードとデータを読んで、x_eq が実際にどう配線されているかの事実を表で出す**タスク。
仮説が正しいかを尋ねているのではなく、コードが実際に何をしているかを報告してほしい。
仮説と食い違う結果でも、そのまま事実を出すこと。

## レビュー手順（必須）

`REVIEW_PROTOCOL.md` に従う。特に以下。

```text
- カバレッジ宣言を冒頭に置く（読んだ/読まなかったファイル・シート、理由）。
- 所見ごとに検証強度を明示：再導出 / 出力照合 / 構造確認。
- 語彙は「下層に支持される / 支持されない / 判断材料不足」。
- 採用/保留/撤回は決めない。補正式はフィットしない。
```

## 背景（疑っていること）

前回レビュー(`REVIEW_CHF_singletube_bundle_xeq_Fform_20260617.md`)で、
単管の x_eq は qP_F1（予測CHF）由来で、応答 PM_F1=qP_F1/qM と qP_F1 を共有する循環があると分かった。

そこで新たに気づいた、**まだ検証されていない第三の軸**がある：

```text
疑い：
  単管では、noF1 と F1 で「違う x_eq」を使っているのではないか。
    noF1 → qP_noF1 由来の x_eq
    F1   → qP_F1   由来の x_eq
    （モデルごとに x_eq を作り直している）

  バンドルでは、noF1 と F1 の両方で「noF1 由来の x_eq」を使っているのではないか。
    noF1 → qP_noF1 由来の x_eq
    F1   → qP_noF1 由来の x_eq（F1でも noF1 のx_eqで固定）

もしこれが本当なら：
  - 単管F1 は x_eq(qP_F1) と PM_F1(qP_F1/qM) が同じ qP_F1 を共有する「最も強い循環」。
  - バンドルF1 は x_eq(qP_noF1) と PM_F1(qP_F1/qM) で、共有は qP_noF1 までの「緩い循環」。
  → 「単管で x_eq が効く／バンドルで弱い」という差の一部が、
     物理の差ではなく x_eq の定義の不一致（配線の差）で説明されてしまう。
     比較する前に、両側で x_eq の定義を揃えるべきだった可能性がある。
```

## やってほしいこと

### Step 1: x_eq の出所を追跡（再導出 / コードを読む）

単管・バンドルそれぞれで、各モデルの x_eq が**どの予測値・測定値から計算されているか**を、
実コード（`computeXeq` 等の呼び出し、入力xlsx を作るスクリプト）を読んで特定する。
推測でなく、コードの該当行を引用して示すこと。

確認対象：

```text
単管：
  - noF1 で使う x_eq は、qP_noF1 / qP_F1 / qM のどれから作られているか
  - F1   で使う x_eq は、qP_noF1 / qP_F1 / qM のどれから作られているか
  - これらは同じ x_eq か、モデルごとに作り直しているか

バンドル：
  - noF1 で使う x_eq は何由来か
  - F1   で使う x_eq は何由来か
  - 両方とも noF1 由来で固定されているか
  - バンドル入力(v1/v2)を作る段階で x_eq がどう書き込まれたかも追う
    （current_bundle_input を作るスクリプト／元シートの x_eq 列の出所）
```

### Step 2: 配線表を出す（事実のみ）

```text
| 系 | モデル | x_eqの由来(qP_noF1/qP_F1/qM) | 応答PMの分子 | 共有ドライバ | 循環の強さ | 根拠コード行 |
| 単管   | noF1 | ? | qP_noF1 | ? | ? | file:line |
| 単管   | F1   | ? | qP_F1   | ? | ? | file:line |
| バンドル | noF1 | ? | qP_noF1 | ? | ? | file:line |
| バンドル | F1   | ? | qP_F1   | ? | ? | file:line |
```

「循環の強さ」は、x_eq と PM が同じ予測値を共有しているか（強）、
F1係数やqMを挟んで間接的か（弱）、共有なし（qM由来＝なし）で区別する。

### Step 3: 単管とバンドルで定義が一致しているか判定

```text
- 単管とバンドルが同じ定義の x_eq を使っているか／いないか。
- 不一致なら、その不一致が前回レビューの「単管で効く／バンドルで弱い」の
  差にどれだけ寄与しうるかを、定性的に述べる（断定はしない）。
```

### Step 4: 不一致だった場合のみ — 定義を揃えて再計算（再導出）

Step 3 で単管とバンドルの x_eq 定義が食い違っていた場合、
**両側を同じ定義に揃えて**、前回の主要量を出し直す。揃える定義は2通りとも出す：

```text
(a) 両側とも「測定パワー qM 由来」の x_eq（循環なしの基準）
(b) 両側とも「noF1 由来 qP_noF1」の x_eq（バンドルに単管を合わせる）

各定義で出す量：
  - 単管：Hsub+P+Tsub 後の x_eq の ΔR² と、残差 vs x_eq の R²
  - バンドル：PM_F1 vs x_eq の R²、Tsub に対する x_eq の追加説明力
  - 両側を同じ定義で見たとき、「単管で効く／バンドルで弱い」差は残るか、消えるか
```

定義を揃えても差が残るなら「単管とバンドルの差」は配線由来ではない。
消えるなら、その差は x_eq の定義不一致が作っていた見かけ、という切り分けになる。
**結論は出さず、揃える前後の数値の動きだけを表で示す。**
