## 5. 一次解釈

### 5.1 FformLinear化で qP / PM は上がる

FformLinearはlegacyより小さくなったため、今回のマクロ実装では qP が上昇した。

これは、マクロ内でF_formが qP に対して実質的に「割る側」に効いているためと考えられる。

### 5.2 noF1

noF1では以下の変化になった。

```text
108:
  PM 0.621831 → 0.653049
  ΔPM = +0.031218

161:
  PM 0.620980 → 0.620980
  ΔPM = 0

164:
  PM 0.570923 → 0.597862
  ΔPM = +0.026939
```

noF1は全体としてまだ過小評価であるが、108と164は少し上がった。

### 5.3 F1

F1では以下の変化になった。

```text
108:
  PM 1.06679 → 1.123223
  ΔPM = +0.056433

161:
  PM 0.908841 → 0.908841
  ΔPM ≒ 0

164:
  PM 0.892018 → 0.939561
  ΔPM = +0.047543
```

FformLinear化によって、164の過小評価はかなり改善した。
一方、108はもともとやや過大側だったため、さらに過大側へ動いた。

### 5.4 重要な読み替え

BT08以前の「F1後は108がほぼ合い、161/164は低め」という構図は、FformLinear化後には以下に変わる。

```text
F1 + FformLinear_v1:
  108 = 1.123  → 過大側
  161 = 0.909  → 低めのまま
  164 = 0.940  → かなり改善
```

したがって、Fform legacyの暫定処理は、特に164の低め残差を一部作っていた可能性がある。
ただし、108は悪化するため、FformLinearだけで全体が整理されたとは言えない。

```text

次：

```
