## 1. 目的

TM03B0で作成した `tm_input_preview`（main280点をmacro `tm` 投入形式に変換した表）の**変換が正しいか**を、既計算済み点の再投入で確認する。

確認経路（既存tm行のコピーではない）:
```
T&M正本MD → TM02 main280 → TM03B0 tm_input_preview
  → xlsmコピーの tm 末尾へ新規行投入（preview由来の値） → VBA/Solver再計算 → 旧計算結果と比較
```
preview入力値だけを使い、すでに旧計算結果がある点を再投入して同じ値に戻るかを見る。これでTM02/TM03B0の新しい280点入力経路が既存計算系と整合するかを判定する。

**本runで行わないこと（厳守）:** 既存xlsmの上書き／直接編集保存をしない。必ずコピーで作業。既存tm行のコピー再投入はしない（投入値は必ず `tm_input_preview` 由来）。280点一括投入はしない。F1再fit／PM補正式／L-D補正式は作らない。

## 5. xlsmコピーへの投入方法

- 元xlsmを `TM03B1a_macro_reinject_validation_20260625_083115.xlsm` に**コピー**して作業（元は未改変）。
- `tm` シート末尾（**226〜233行**）へ8点を新規行として追加。Excelテーブル `テーブル2` のrefを `A1:BR225 → A1:BR233` に拡張。
- **入力21列**（A,B,M,N,Q,R,S,T,V,X,AC,AG,AP,BE,BG,BH,BI,BJ,BK,BQ,BR）は **preview由来の値**を投入（M=Noのみ連番225..232を継続付与）。
- **数式49列**（C–L物性, O=A, P=Pw, U,…,BB=q_P, BF=PM_ratio, BL=Tsub 等）は、取込み実績行(88行)の数式を openpyxl `Translator` で各新規行へ相対参照シフトして付与。
  - openpyxl環境では Excel の「テーブル自動展開」が走らないため、数式は明示的に複製した（自動展開に相当する結果を確実化）。**自動展開されなかった列はない**（49列すべて複製済み）。
  - 注記: 物性列 C–L は元は1セル配列数式（CSE）だが、スカラを返すUDFのため通常数式として書き込んだ。Excelで開くと再計算される想定（要・開封時確認）。構造化参照 `テーブル2[[#This Row],…]` と A1相対参照の混在式も正しく変換済み（例 `U226 = =+T226+BG226*S226*テーブル2[[#This Row],[Pw]]*R226/(N226*O226*K226)`）。
- 投入後検証: 入力値一致、数式の行参照正、VBA 33モジュール（Module6含む）健在、`テーブル2` ref=A1:BR233、234行以降は空。

## 10. 不一致がある場合の原因候補（事前整理）

実行後に許容差を超えた場合の優先疑い:
1. **Table10 の Tin差(0.25–0.36K)**: 旧Table10 Tinが非IF97ソース。q_Pがずれるなら最有力。→ 旧Table10 Tinの算出法を確認。
2. **SatProp Tsat と IF97 Tin の組合せ**: BL Tsub=Ts(SatProp)−Tin(IF97) の微小不整合（TM03B0 §7既出）。
3. **q_in seed の扱い**: 本投入では S=q_M をseed。マクロ外側二分探索は bounds(1e5–1.2e7) の中点でSを上書きするため seed値は無関係のはず（要Log確認）。
4. **固定列(F_form/Fcorr/F2/A_corr/σ_corr)**: 全て legacy と同値(1/1/1/0.046/5625)で投入済み。差異なし想定。
5. **F1なし/F1あり取り違え**: 主比較は `ST_noF1`。F1あり値と比べない。
6. **旧結果側の列対応違い**: 70列同一構造のため低リスク。
7. **Solver収束条件/初期値**: マクロが毎行 f=0.01/Tw=600/y*=1e-5/UB=1 に初期化。旧計算と同条件。
8. **マクロ未実行行との比較**: 旧 `ST_noF1` は実行済み（q_P≠q_in seed, dq_ratio≈数%）であり未実行行ではない。
