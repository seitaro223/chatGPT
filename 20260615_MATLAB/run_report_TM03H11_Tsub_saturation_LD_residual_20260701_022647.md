# TM03H11: Table10 lowX scope B — Tsub飽和型補正とL/D残差診断

作成時刻: 20260701_022647

## 目的

TM03H10で仮固定した Table10 source01 lowX の主候補 `G<=4200, L/D>=40` を対象に、Tsub補正の関数形と、Tsub補正後に残るL/D方向残差を確認した。

本作業の目的は、最高R2の式を採用することではなく、以下を切り分けることである。

- Tsub 2次関数は数値的には合うが、物理的に説明しにくい高Tsub側低下を含むか。
- Tsub効果を指数関数型・飽和型で表せるか。
- Tsub飽和補正後にL/D方向の残差が残るか。
- xは状態分類・診断量として効くか。ただし補正式入力にはしない。

## 対象データ

- 入力: `/mnt/data/TM03H10_Table10_lowX_adoption_scope_20260630_235447/TM03H10_scope_B_candidate_records_20260630_235447.csv`
- 対象: scope B = `G<=4200` and `L/D>=40`
- N = 113
- `x<=-0.20`: N = 10
- `L/D>=120`: N = 9
- `PM>=2.2`: N = 2

## Tsub飽和型モデル

採用候補として見た飽和型は以下。

```text
PM = a + b * (1 - exp(-Tsub/c))
```

fit結果:

```text
a = -0.584563
b = 2.420680
c = 63.272074 K
asymptote = a+b = 1.836117
R2 = 0.844372
```

2次関数よりR2は少し低いが、Tsub効果を「低Tsub側で強く効き、高Tsub側で飽和する」と説明できるため、物理的にはこちらの方が自然である。

## Tsub飽和型 + L/D診断

L/D方向を、まず補正式ではなく残差診断として入れたモデルは以下。

```text
PM = a + b * (1 - exp(-Tsub/c)) + d * log((L/D)/80)
```

fit結果:

```text
a = -0.340236
b = 2.436659
c = 90.332805 K
d = -0.296472
asymptote at L/D=80 = a+b = 2.096424
R2 = 0.902025
```

`d` は負であり、Tsub飽和型でならした後、高L/D側はP/Mが下がる方向に整理された。

目安として、L/D=80基準から見ると、

```text
L/D=120: ΔPM ≈ -0.120
L/D=300: ΔPM ≈ -0.392
```

である。

この結果は、ユーザーコメントの「2次関数の高Tsub側低下は、TsubではなくL/D影響をTsubが代表しているように見える」という読みと整合する。

## 残差診断

Tsub飽和型だけで補正した後の残差は、L/Dおよびxと対応した。

- saturation residual vs log(L/D): R2 = 0.210
- saturation residual vs x: R2 = 0.555

L/Dを入れると、log(L/D)方向の残差はほぼ消えた。一方で、x方向の残差はまだ残る。

ただし、xはcurrent record上の熱平衡クオリティであり、qM由来の結果側情報を含む。したがって、xを入れたモデルはR2が非常に高くなるが、これは補正式候補ではなく、状態分類・診断として扱う。

## 判断

1. Tsub 2次関数は数値的には有効だが、高Tsub側で下がる形をTsubの直接効果として読むのは難しい。
2. Tsub効果は、指数関数型・飽和型で読む方が物理的に自然である。
3. Tsub飽和補正後、L/D方向の残差が残る。これは、L/Dまたは加熱履歴・沸騰履歴長の代理として見る価値がある。
4. ただし、L/Dをただちに補正式にするのではなく、まずは残差診断項として扱う。
5. xを入れると説明力は非常に高いが、xは結果側情報を含むため、補正式入力にはしない。
