# TM03H Table10 quality / Tsub diagnostic

作成日時: `20260629_145410`

## 1. 目的

TM03F2で、Table10はG範囲が広いものの、`1900 <= G <= 4200` や `2400 <= G <= 4200` にそろえても `PM_noF1 ~ Tsub` の相関が明確に改善しなかった。

本runでは、Table10の違和感を追う第一段として、現在の `main280/current_result` に含まれる **Table10 source01 lowX valid点** を対象に、Tsub、G、x_Mes（Exit quality / 熱平衡クオリティ相当の診断量）、qM、L/D のどれが `PM_noF1` と強く対応しているかを確認する。

補正式は作らない。R2が高いモデルを採用しない。特に `x_Mes` および `qM` は実験結果側の熱収支量を含む可能性があるため、**診断量**として扱う。

## 9. 一次判断

今回の一次判断は以下。

```text
1. Table10でTsub相関が弱めに見える理由は、G範囲だけでは説明しにくい。

2. Table10では、PM_noF1はTsub単独よりx_Mesと強く対応する。

3. Gを1900〜4200や2400〜4200に制限しても、Tsub相関は明確には改善しない。

4. 一方、x_Mesとの対応はG帯別に見ても強く残る。

5. したがって、昔Tsub相関がきれいに見えたのは、x_Mes/熱平衡クオリティ側の整理を十分に分けていなかったため、Tsubが熱収支状態量の代理に見えていた可能性がある。
```

ただし、x_MesやqMは結果側の量を含む可能性があるため、補正式入力として使うとは言わない。

## 10. 次に見るべきこと

次にTable10を追うなら、以下を確認する。

```text
T10H2:
  Table10で、x_Mesを実験側品質として見るだけでなく、
  前向き計算で使えるx_eqまたは予測側熱収支量が作れるか確認する。

  目的：
    x_Mes相関が、単なるqM由来の結果側相関なのか、
    前向き計算で使える熱収支状態量の候補なのかを分ける。
```
